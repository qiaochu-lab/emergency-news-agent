# 泛应急技术行业周报模板（研报化版本）

> 目标：参考证券行业周报的结构、语气和结论组织方式，将多源公开信息整理成“可读、可跟踪、可决策”的行业研究型周报。
>
> 使用原则：
> - 先结论，后展开
> - 先周度景气/主题变化，再重点新闻，再事件日历，再风险提示
> - 每条重要信息尽量给出时间、主体、影响和后续观察点

**生成时间：** YYYY-MM-DD HH:MM:SS

# 报告要点

> 用 3-5 条 bullet 先把整篇报告最重要的判断说清楚，写法参考券商周报首页摘要。

- `周度景气判断`：本周泛应急技术相关主题整体表现如何，哪些方向景气度仍在延续，哪些方向出现回调或分化。
- `核心主题判断`：本周最值得关注的 1-2 个技术/产业主题，例如 AI 应用、5.5G/5G-A、卫星互联网、低空与无人系统、应急通信体系升级等。
- `重点事件判断`：选取最有代表性的 1-3 条事件，说明为什么值得重视。
- `建议关注方向`：下一阶段建议持续跟踪的技术链条、场景方向、政策进展或产业节点。
- `风险提示`：本周判断可能失效的主要风险因素。

# 1 周度景气与板块变化

## 1.1 行业整体表现

> 对本周整体表现做一句结论，再给出 2-4 条支撑。

本周泛应急技术相关主题整体呈现 XXX，主因是 XXX。

- 大类主题表现：AI / 通信 / 无人机 / 泛应急分别如何变化
- 主要驱动因素：政策、产业发布、公司事件、技术突破、海外映射等
- 市场情绪：偏强 / 分化 / 回调
- 本周结论：哪些方向景气延续，哪些方向需谨慎观察

## 1.2 细分方向表现

> 模仿券商“三级子行业”写法，突出“哪个方向最强、哪个方向最弱、为何如此”。

本周细分方向中，`上涨 / 热度提升` 最明显的是 XXX，`回调 / 热度下降` 最明显的是 XXX。

建议按以下方向拆解：

- AI 应用 / Agent / 多模态
- 算力基础设施 / 网络设备 / 数据中心链条
- 5G-A / 6G / RedCap / 专网
- 卫星互联网 / 北斗 / NTN
- 无人机 / 低空系统 / 自主飞行
- 应急指挥 / 灾害感知 / 韧性通信

## 1.3 重点主体 / 重点事件表现

> 不一定写“个股涨跌幅”，但要保留证券研报中“谁最强、谁最值得解释”的信息组织方式。

本周重点主体或重点事件层面，建议列出 3-5 个最有代表性的对象：

- 事件 / 公司 / 机构 1：一句话说明本周表现或动作
- 事件 / 公司 / 机构 2：一句话说明本周表现或动作
- 事件 / 公司 / 机构 3：一句话说明本周表现或动作

# 2 本周重点新闻与分析

> 这一部分是正文核心，建议保留 3-6 条最值得写的事件。每条都按“事实 + 研究判断”写，语气参考行业周报。

## 2.1 标题一

**事件概述：**  
用 2-4 句话客观说明发生了什么，写清时间、主体、动作和结果。

**分析判断：**  
用 1-2 段说明这件事为什么重要，建议覆盖：

- 对产业链 / 技术路线 / 场景落地意味着什么
- 对泛应急场景的关系是什么
- 是短期催化、中期趋势，还是长期方向
- 还需要继续验证什么

# 4 建议关注方向

> 参考证券周报“建议关注方向 / 推荐标的”的功能，但不强行写股票，改写成“建议持续跟踪的技术链条和场景方向”。

建议关注方向：

- `方向一`：例如算力基础设施 / 5G-A / 卫星互联网 / AI 应用 / 低空安全
- `方向二`：说明为什么值得关注
- `方向三`：说明下一步看什么验证信号

可选写法：

- 技术主线：XXX
- 场景主线：XXX
- 政策主线：XXX

# 5 风险提示

> 保留证券研报结尾风格，简短、明确、非空话。

- 政策推进节奏不及预期风险
- 产业落地节奏不及预期风险
- 市场需求不及预期风险
- 海外环境或供应链不确定性风险
- 技术成熟度与商业闭环不及预期风险

---

# 附：写作风格要求

- 语气应接近“行业研究周报”，不要写成公众号、新闻播报或学术综述
- 小标题要像研报目录，可直接定位
- 每一节开头尽量先给结论，再给展开
- 多用“表明 / 反映 / 说明 / 预计 / 建议关注 / 值得跟踪”等研报语气
- 不要堆砌形容词，不要写成营销文案
- 不要只有事实摘抄，必须有“分析判断”
